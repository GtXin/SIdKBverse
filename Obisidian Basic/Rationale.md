# Obsidian: Core Design Rationales

## Overview

Obsidian was not created to be another note-taking application.

It was created to help individuals build a durable, interconnected body of knowledge that they fully own.

The software is built around several core beliefs.

---

# 1. Knowledge Should Belong to the User

Most modern software stores information inside proprietary cloud platforms.

Obsidian rejects this model.

All notes are stored as local Markdown files that remain accessible regardless of the software being used.

## Principle

> Your knowledge should belong to you, not the platform.

---

# 2. Knowledge is a Network, Not a Hierarchy

Traditional software organizes information using folders.

However, ideas often belong to multiple contexts simultaneously.

Example:

Accommodation may relate to:

- Myopia
- Presbyopia
- Wavefront
- Visual Behavior

A single folder cannot represent these relationships effectively.

Obsidian therefore emphasizes links rather than rigid hierarchies.

## Principle

> Knowledge behaves like a network rather than a filing cabinet.

---

# 3. Ideas are More Valuable Than Documents

Documents are containers.

Knowledge is the content.

Most software focuses on managing documents:

- Files
- Pages
- Folders

Obsidian focuses on preserving and connecting ideas.

## Principle

> The idea matters more than the document that contains it.

---

# 4. Knowledge Compounds

A note has limited value by itself.

Value emerges when multiple ideas become connected.

Example:

Idea A
+
Idea B
+
Idea C
=
New Insight

Obsidian is designed to encourage the creation of these connections.

## Principle

> Knowledge becomes more valuable as connections increase.

---

# 5. Thinking is More Important Than Productivity

Many productivity tools optimize for:

- Tasks
- Checklists
- Deadlines
- Project Tracking

Obsidian was designed primarily as a thinking environment.

It aims to support:

- Learning
- Reflection
- Understanding
- Idea Generation

## Principle

> Better thinking leads to better decisions.

---

# 6. Long-Term Knowledge is More Valuable Than Short-Term Projects

Projects eventually end.

Knowledge persists.

A project may disappear, but the lessons, concepts, and understanding developed during the project remain valuable.

Obsidian therefore focuses on preserving knowledge that can survive beyond individual projects.

## Principle

> Build knowledge assets, not just project archives.

---

# 7. Small Core, Infinite Extensions

The founders intentionally kept the core application simple.

Instead of building every feature themselves, they created a plugin architecture that allows users to extend the software for their own needs.

## Principle

> The tool should adapt to the user, not the user to the tool.

---

# One-Sentence Summary

Obsidian exists to help people build a permanent, interconnected body of knowledge that they own completely and can continue developing for decades.
